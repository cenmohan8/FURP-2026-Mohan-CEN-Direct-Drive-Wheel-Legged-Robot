## Blackwell(50 系)三个必须避开的坑

这是你和普通 40 系用户最大的区别,装之前先记死:

**坑 1:驱动必须用 "open" 开源内核模块变体。** 专有驱动分支不支持 RTX 50 系,装的时候要选 "open" 变体,或者用 .run 安装器时选 MIT/GPL 开源模块选项。另外最常见的错误是在旧驱动上直接装新驱动,一定要先清干净再装。

**坑 2:PyTorch 不能用 Isaac Sim 自带的旧版,必须换 nightly + cu128。** 对 50 系显卡,要用最新的 PyTorch nightly 构建,而不是 Isaac Sim 自带的 PyTorch 2.5.1:

```bash
pip install --upgrade --pre torch torchvision --index-url https://download.pytorch.org/whl/nightly/cu128
```

讨论帖里 5080、5060Ti 的用户都验证了这条命令能跑,装完忽略那些不兼容报错就行,能正常工作。

**坑 3:驱动版本别上太新,有 PhysX 兼容问题。** 这是最新的坑——有 5070 用户用 580 系驱动会遇到 PhysX GPU 直接掉回 CPU 跑,解决办法是把驱动降到 570.195.03 这一档,降级后就正常了。所以你装的时候建议优先用 R570 production 分支驱动,别盲目上最新的 580。

这份 README 是**双臂移动机械臂组**的项目说明书,核心一句话:**让一台装在"转向轮底盘"上的双臂机器人,把底盘和两条胳膊当成一个整体来控制。**

它的逻辑链是这样的:

普通固定机械臂只能够到胳膊长度范围内的东西。把胳膊装到能移动的底盘上,工作范围理论上就无限大了——但代价是**底盘和胳膊的运动会互相耦合**,变成一个"自由度过多、有冗余"的系统。这个系统的数学核心是**整体雅可比矩阵 J**,它把底盘、左臂、右臂三部分对末端执行器的贡献堆在一起,描述"你想让手怎么动 → 各个关节该怎么转"。因为自由度比任务需要的多,就有了**零空间**,可以拿来干副业(比如让关节别顶到极限、避免自撞)。

文档把工作分成清晰的阶段,从易到难:

1. **MoveIt 入门**(第一阶段,纯练手,不做研究)——把机器人模型在 RViz 里跑起来、配好 MoveIt、让单臂/双臂做点到点运动、用代码而不是界面去控制。
2. **单臂控制**——一条胳膊的笛卡尔/关节控制,先仿真后上真机。
3. **双臂协调**——两条胳膊一起抓一个东西,保持固定的相对姿态(就是那个 `J_rel q̇ = 0` 约束,意思是搬同一个物体时两只手的相对位置不能变)。
4. **整体 QP 控制**——把底盘+两臂当一个系统,写成一个**二次规划(QP)**,每个控制周期解一次。QP 的好处是能把"任务目标""关节限位""转向约束""自撞避免"全塞进去当约束,解出来的就是安全的关节指令。
5. **强化学习增强**——用 RL 学一个"残差策略"叠在 QP 上面,补偿那些难建模的东西(轮子打滑、接触、负载变化)。关键设计是 **QP 一直留在回路里**,所以哪怕 RL 策略还没学好,安全约束也永远被强制执行。

剩下的部分就是配套:转向轮底盘的麻烦(打滑、里程计漂移、转向角约束)、建模要识别哪些参数、安全规则(随手能按的急停、低速起步、人离开运动范围)、评估指标(末端跟踪误差、QP 求解时间、sim-to-real 差距)等等。

**一句话总结这文档和你的关系**:它是隔壁双臂组的"模型驱动控制(MoveIt + QP)+ RL 当辅助"路线;你 hoppertrex 组是"纯 RL(Isaac Lab)让机器人站起来"路线。两条线的共同点只有"最后都会用到 RL",但起手式完全不同——他们先 MoveIt,你先 Isaac Lab。所以你别被这文档带偏去装 MoveIt,你的第一步就是上面那套 Isaac Lab。

## 第一大块:装 Ubuntu 22.04 双系统

### 需要的东西
- 一个 **≥8GB 的 U 盘**(会被格式化,里面东西先拷走)
- 你的笔记本(已经有 Windows)

### 官方下载链接(直接甩你脸上)

1. **Ubuntu 22.04 LTS 桌面版镜像下载**:https://releases.ubuntu.com/22.04/
   - 下那个叫 `ubuntu-22.04.x-desktop-amd64.iso` 的文件(x 是小版本号,选最新的就行,大概 4-5GB)
2. **Rufus(把镜像写进 U 盘的工具,Windows 上用)**:https://rufus.ie/

### 步骤

**1. 做启动 U 盘**
下好 ISO 和 Rufus 后,插上 U 盘,打开 Rufus → "设备"选你的 U 盘 → "引导类型选择"点 SELECT 选刚下的那个 ISO → 分区类型选 **GPT** → 目标系统选 **UEFI** → 点 START。它可能问你用 ISO 模式还是 DD 模式,选推荐的(ISO)就行。等它写完。

**2. 在 Windows 里腾出空间给 Ubuntu**
这步是给 Ubuntu 准备一块空地。Win 键搜"创建并格式化硬盘分区"→ 打开磁盘管理 → 右键你空间最大的那个盘(通常是 C 或 D)→ "压缩卷" → 输入要分给 Ubuntu 的大小。

**⚠️ 给多少:Isaac Lab 加 Isaac Sim 非常占地方,本体加缓存轻松上百 GB,强烈建议至少留 150-200GB(够的话 250GB 更舒服)。** 输入的数字单位是 MB,比如 200GB 就填 `200000`。压缩完会出现一块"未分配"的黑色区域,**这块别动它,保持未分配,装 Ubuntu 时再用。**

**3. 关掉快速启动 + 进 BIOS 关 Secure Boot**
- Windows 里:控制面板 → 电源选项 → "选择电源按钮的功能" → "更改当前不可用的设置" → 取消勾选"启用快速启动" → 保存。
- 重启,开机狂按进 BIOS 的键(联想常是 F2/Fn+F2,有些是 F1、ESC、DEL,你这台是什么牌子告诉我我帮你确认)。在 BIOS 里把 **Secure Boot 设为 Disabled**。**注意:50 系显卡用 open 驱动时 Secure Boot 开着会很麻烦,这里关掉省后患。** 保存退出。

**4. 从 U 盘启动,装 Ubuntu**
- 插着 U 盘重启,开机按启动菜单键(常是 F12 / F10 / ESC,联想有时是那个小针孔 Novo 键),选从你的 U 盘(带 UEFI 字样的那个)启动。
- 进去选 "Try or Install Ubuntu" → 选 Install → 语言键盘随你 → 联网那步可以连 WiFi → "Updates and other software"那页勾上 **"Install third-party software..."**(这个跟显卡驱动有关,勾上)。
- **到了最关键的"Installation type"这一页,千万看清:**
  - 选 **"Install Ubuntu alongside Windows Boot Manager"**(和 Windows 共存)——如果有这个选项,选它最省心,它会自动用你刚才那块未分配空间。
  - **⚠️ 千万不要选 "Erase disk and install Ubuntu"——那个会把你整个硬盘包括 Windows 全部抹掉!** 这是双系统唯一的致命错误,看到 "Erase" 字样就停下来。
- 后面设时区、用户名密码,等它装完,提示重启时**拔掉 U 盘**再回车。

**5. 验证**
重启后应该会出现一个黑底的开机菜单(GRUB),能选 Ubuntu 或 Windows Boot Manager。能两个都进得去,双系统就成了。

> 如果重启后直接进了 Windows、没看到选择菜单:多半是启动顺序问题,进 BIOS 把 "ubuntu" 调到启动项第一位即可。卡这步就把 BIOS 界面拍给我。

## 第二大块:装 NVIDIA 驱动(进了 Ubuntu 之后做)

现在你已经在 Ubuntu 桌面里了。先确认系统能上网(右上角连 WiFi)。

### 思路
Ubuntu 自带一个很方便的驱动安装方式,**对大多数人来说根本不用去手动 purge、不用跑 .run 安装器**——那套手动流程(链接 4 那个 gist)是给装失败、要从头清干净的人兜底用的,你第一次装**先试最简单的方法**,成了就别碰复杂的。

### 方法 A(首选,最简单):用 Ubuntu 的图形化驱动工具

1. 打开终端(Ctrl+Alt+T),先更新一下源:
   ```bash
   sudo apt update
   sudo apt upgrade -y
   ```
2. 打开左下角"显示应用程序",搜 **"Software & Updates"(软件和更新)**,打开它,切到最右边的 **"Additional Drivers"(附加驱动)** 标签页。
3. 它会列出可用的 NVIDIA 驱动。**优先选带 "open" 字样、版本号 570 那一档**(比如显示 `Using NVIDIA driver metapackage from nvidia-driver-570-open`)。50 系必须用 open 变体,这点前面讲过。选中 → 点 "Apply Changes" → 等它装完。
4. 装完 **重启**。

5. 重启后开终端验证:
   ```bash
   nvidia-smi
   ```
   能看到一张表,里面有 `NVIDIA GeForce RTX 5070 Laptop GPU`、驱动版本(应该是 570.x)、CUDA 版本,就成了。

### 如果 Additional Drivers 里没有 570-open 选项

那就用命令行装(还是很简单,不用 purge):
```bash
sudo add-apt-repository ppa:graphics-drivers/ppa
sudo apt update
sudo apt install nvidia-driver-570-open
sudo reboot
```
重启后同样 `nvidia-smi` 验证。

### 方法 B(兜底):只有装坏了、要从头清干净才用

只有当你之前装过驱动、现在 `nvidia-smi` 报错或者黑屏循环登录,才需要先 purge 再重装。这就是链接 4 那个 gist 的场景。**第一次干净安装的你大概率用不到,所以先别跑这些命令:**

```bash
# 仅在驱动装坏需要清干净时才用:
sudo apt-get remove --purge '^nvidia-.*'
sudo apt autoremove
sudo reboot
```
purge 之后再回到方法 A 重装。链接 4 的完整兜底指南:https://gist.github.com/jatinkrmalik/86afb07cbe6abf5baa2d29d3842aa328

## 你现在按这个顺序走就行

1. 备份 Windows 重要文件
2. 下 Ubuntu 22.04 ISO + Rufus,做启动 U 盘
3. Windows 磁盘管理压缩出 ≥150GB 未分配空间
4. 关快速启动、进 BIOS 关 Secure Boot
5. U 盘启动,装 Ubuntu,**认准 "alongside Windows",躲开 "Erase disk"**
6. 进 Ubuntu,Software & Updates → Additional Drivers → 装 **570-open**
7. 重启,`nvidia-smi` 验证

走完这两大块,你就有了一个能跑 Isaac Lab 的干净环境,然后才接到我上一条说的 PyTorch nightly cu128 那步。
